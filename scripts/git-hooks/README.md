# Git-хуки безопасности

Защита от повторения инцидента с утечкой дампов БД (30.06.2026): дампы попали в
**публичный** репозиторий через `git add -f` (в обход `.gitignore`).

- **pre-commit** — блокирует коммит дампов БД (`*.sql`, `*.sql.gz`, `*.dump`),
  файлов `backup-*` и секретов (`.env`, `.env.*`; кроме `.env.example`).
  Срабатывает даже на `git add -f`.
- **pre-push** — последний барьер: сканирует **всю** отправляемую историю, ловит
  файл, даже если он был добавлен и удалён внутри тех же коммитов.

## Активация (нужно один раз на каждый клон)

```bash
git config core.hooksPath scripts/git-hooks
```

Обход в исключительном случае: `git commit --no-verify` / `git push --no-verify`
(крайне не рекомендуется — именно так и утекают данные).
